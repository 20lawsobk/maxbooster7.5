# Information Security Policy

**Document Version:** 1.0  
**Effective Date:** November 11, 2025  
**Last Reviewed:** November 11, 2025  
**Policy Owner:** Chief Information Security Officer  
**Compliance Standards:** ISO 27001 (A.5), SOC2, GDPR

## 1. Purpose

This Information Security Policy establishes the framework for protecting Max Booster's information assets, including data, systems, networks, and infrastructure. This policy ensures compliance with SOC2, ISO 27001, and GDPR requirements.

## 2. Scope

This policy applies to:
- All employees, contractors, and third parties with access to Max Booster systems
- All information assets owned or processed by Max Booster
- All locations where Max Booster conducts business operations

## 3. Information Security Objectives

1. **Confidentiality**: Ensure information is accessible only to authorized individuals
2. **Integrity**: Maintain accuracy and completeness of information and systems
3. **Availability**: Ensure authorized users have access when needed

## 4. Information Security Principles

### 4.1 Access Control
- All access to information systems must be authorized
- Least privilege principle applies to all user accounts
- Multi-factor authentication (MFA) required for administrative access
- Regular access reviews conducted quarterly

### 4.2 Data Protection
- Data classified according to sensitivity levels (Public, Internal, Confidential, Restricted)
- Encryption required for data in transit (TLS 1.3) and at rest (AES-256)
- Data retention policies enforced based on legal and regulatory requirements
- Secure disposal of data when no longer required

### 4.3 Physical and Environmental Security
- Restricted access to data centers and server rooms
- Environmental controls (temperature, humidity, fire suppression)
- Regular facility security assessments

### 4.4 Operations Security
- Change management procedures for all system modifications
- Capacity management to ensure system availability
- Backup and recovery procedures tested quarterly
- Security monitoring and logging enabled on all systems

### 4.5 Communications Security
- Network segmentation to isolate sensitive systems
- Secure communications protocols required (HTTPS, SSH, TLS)
- Regular vulnerability assessments and penetration testing
- Firewall rules reviewed and updated regularly

### 4.6 System Development and Maintenance
- Secure development lifecycle (SDLC) practices
- Security requirements integrated into all development projects
- Code reviews and security testing before production deployment
- Regular security updates and patch management

### 4.7 Supplier Relationships
- Due diligence on all third-party service providers
- Data Processing Agreements (DPAs) required for data processors
- Regular security assessments of critical suppliers

### 4.8 Incident Management
- Security incidents reported immediately
- Incident response plan activated for confirmed incidents
- Root cause analysis and lessons learned documented
- Breach notification procedures per GDPR requirements (72 hours)

### 4.9 Business Continuity
- Business Impact Analysis (BIA) reviewed annually
- Disaster Recovery Plan tested semi-annually
- Redundancy and failover capabilities for critical systems

## 5. Roles and Responsibilities

### 5.1 Executive Management
- Approve information security policy and resources
- Support security initiatives across the organization
- Review security metrics and audit findings quarterly

### 5.2 Information Security Team
- Implement and maintain security controls
- Monitor security events and respond to incidents
- Conduct security awareness training
- Perform risk assessments and audits

### 5.3 All Employees
- Comply with information security policies
- Report security incidents and concerns
- Complete mandatory security awareness training
- Protect credentials and access privileges

## 6. Policy Compliance

### 6.1 Enforcement
- Violations may result in disciplinary action up to and including termination
- Compliance monitored through audits, assessments, and automated controls

### 6.2 Exceptions
- Exception requests must be documented and approved by CISO
- Compensating controls required for approved exceptions
- Exceptions reviewed quarterly

## 7. Policy Review

This policy is reviewed annually or when significant changes occur to:
- Business operations or organizational structure
- Technology infrastructure
- Regulatory or compliance requirements
- Threat landscape or security incidents
